Requisitos del Sistema

Requisitos Mínimos

  • 4GB de RAM
  • 2 núcleos de CPU
  • 8GB de espacio en disco
  • Ubuntu 20.04/22.04, Debian 10/11, CentOS 7/8

Requisitos Recomendados

  • 8GB de RAM (16GB para más de 100 usuarios)
  • 4 núcleos de CPU (8 para más de 100 usuarios)
  • 50GB+ de espacio en disco (SSD recomendado)
  • Ubuntu 22.04 LTS o RHEL 8/9

Recomendaciones

Seguridad

Configurar firewall, usar HTTPS con certificados válidos, actualizaciones periódicas y backups automatizados.

Base de Datos

PostgreSQL separado del servidor principal para mejor rendimiento. Considerar replicación para alta disponibilidad.

Escalabilidad

Planear crecimiento con antelación. Considerar GitLab Runner en máquinas separadas para CI/CD.

Pasos Básicos

1. Instalar Docker Desktop

Descarga e instala Docker Desktop en Windows desde el sitio oficial. Asegúrate de habilitar la integración con WSL2 si está disponible.

2. Crear carpetas locales

Crea directorios en tu PC para almacenar la configuración, logs y datos de GitLab. Ejemplo: C:\gitlab\.

3. Ejecutar contenedor de GitLab

Corre el contenedor con el comando docker run para iniciar GitLab en tu máquina.

4. Esperar inicialización

GitLab tardará algunos minutos en configurarse dentro del contenedor. Revisa los logs en Docker Desktop para monitorear el progreso.

5. Acceder a GitLab

Abre tu navegador y entra a http://localhost para comenzar la configuración inicial.

6. Configuración inicial

Crea el usuario administrador, define la contraseña y ajusta las opciones básicas para empezar a usar tu instancia de GitLab.

terminal 
# Ejemplo de instalación en Windows usando Docker

Conclusiones

Ventajas

  • Solución todo-en-uno para gestión de código, CI/CD y más
  • Fácil instalación y mantenimiento con paquetes Omnibus
  • Alta escalabilidad para equipos de todos los tamaños
  • Comunidad activa y documentación extensa

Consideraciones

  • Requiere recursos considerables para instancias grandes
  • Curva de aprendizaje para configuraciones avanzadas
  • Mantenimiento regular necesario (actualizaciones, backups)
  • Configuración de alta disponibilidad compleja

Recomendación Final para PMA

El grupo PMA, se recomienda comenzar con una instalación estándar en un servidor con los requisitos recomendados, implementar un plan de backups automatizado y considerar la configuración de integración continua desde el inicio para maximizar los beneficios de GitLab en sus proyectos.
